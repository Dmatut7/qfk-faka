## 待澄清的产品决策(非阻塞,已采用默认值,可随时调整)

> 以下为第一阶段规划 + 对抗式设计评审后做出的默认取舍,已写入 spec(§10.5)。如需调整请告知,否则按默认推进;核心模块(M5/M6)有人工验收点可届时校正。

**架构与流程默认:**
1. **多商户 SaaS 模式**:平台运营、商户入驻。默认成立。
2. **买家游客下单**:支持游客(邮箱+联系方式),非强制账号。
3. **发货时机**:下单即预占卡密(锁 15min),支付成功转已售,超时释放。
4. **金额**:`DECIMAL(10,2)` + bcmath(元)。
5. **结算**:支付成功即按 `commission_rate` 入商户余额,商户提现、平台审核。
6. **并发取卡**:`FOR UPDATE SKIP LOCKED`(本机 MySQL 9.5 支持)。
