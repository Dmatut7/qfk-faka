# 对标鲸发卡(鲸商城PRO)功能差距 + 路线图

> 来源:无头浏览器实抓 `fkdemo.jingsoft.com/shop/bbb` 的渲染页 + 截获其 `shopApi` 真实数据模型(2026-06-21)。
> 标记:✅已有 / 🟡部分 / ❌缺失。每项标优先级 P0–P3。

## 一、店铺橱窗层(店铺页 /shop/{slug})
| 功能 | 鲸发卡(实抓)| 我们 | 优先级 |
|---|---|---|---|
| 店铺**头像 + 封面横幅** | avatar + 主题背景图 | ❌ | P0 |
| 店铺**简介** | description「期待为您服务」| 🟡 仅店名 | P0 |
| **保证金** | deposit_money 10190.88,显著展示 🛡 | ❌ | P0 |
| **已认证**徽章 | 真实字段 | 🟡 前端写死 | P0 |
| **成交数** | sell_count 200 | 🟡 有数据不展示 | P0 |
| 登录**归属地** | login_province 深圳市(反诈信号)| ❌ | P2 |
| **客服多渠道** | kefu:QQ/微信/企业微信/手机/二维码/邮件 + 自定义客服状态 | ❌ 死按钮 | P0 |
| **商品类型 tab** | goods_type:卡密/知识/资源/权益 | ❌ 只有自动卡密 | P1 |

| **商品封面图** | image | ❌ 仅 emoji | P0 |

| **限购** | limit_count | ✅ max_buy | — |
| **店铺主题/装修** | shop_pc_style/h5_style + theme_params(默认/助力/知乎/全屏/花瓣 多模板+背景)| ❌ | P2 |
| 搜索 | 顶部搜索 | ❌ | P1 |

## 路线图(按"小改动大体验"排序)
- **P0 店铺橱窗地基**(让店铺像真站):商品图 + 划线价;店铺装修字段(封面/头像/简介/公告/客服 QQ微信手机/保证金/认证);分类带图+商品数+买家端可筛;店铺页展示成交/保证金/认证/客服;**2 列带图商品网格**;**丰富演示数据**(多商品/多分类/带图)。
- **P1 信任+营销+类型**:商品类型(卡密/知识/资源/权益);优惠券;订单查询密码;联系方式格式(QQ/手机/邮箱);商品搜索;富文本详情;公告系统;商户自助入驻。
- **P2 深度**:库存模糊显示;折扣/多件优惠/满赠;店铺多主题;商品图文;安全提示;归属地;备案/SEO。
- **P3 生态**:推广分销;多资金模式;评价信誉。

## 同步并入的缺陷(来自审计 wdhj9jhrl,见 improvements.md 待办池)
支付方式造假、订单只显示#id、补发无确认、渠道密钥明文、type=2 卖不了、异常单无退款、对账时间口径。
