## ⚠️ 需后端配合 / 需你拍板(我的工程判断:前端单方做不彻底)
- **数据口径**:今日毛利=成交额、佣金费率 0%、提现手续费恒 0、平台利润 0 —— 多半是演示种子未配成本/费率,或聚合漏算。需后端/你确认是否补成本与费率。
- **统计卡全局聚合**:商户审核/跨商户订单/投诉仲裁/提现 的统计卡现为"本页 reduce"口径,翻页/筛选即失真。需后端返回各状态总数(by_status/total)前端才能正确显示。
- **真实支付二维码**:现为占位图(已诚实化文案)。要真扫码需后端返回二维码内容 + 前端引 qrcode 库。
- **图片上传**:商品主图 / 店铺 logo·封面 现仅支持粘 URL。需后端上传接口 + 图床。
- **资源类"重新获取下载链"+ 倒计时**:需后端刷新签名链接口。
- **权益类"已使用/已过期"态**:需后端返回每条权益码状态字段。
- **部分退款 / 强制下架违规商品 / 商品主图必填**:需后端能力或产品决策。
- **多支付渠道(微信/支付宝/USDT 大额)**:现后端为 epay 单聚合,Kit 要求多渠道;需确认是降级还是补渠道。
