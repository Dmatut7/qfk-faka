## 3. 业务逻辑层关键修复(测试测不出来的设计缺陷)
- **卡密一卡一售/spec§10.5**:退款不再把已发货(SOLD)卡密回库重售(原会同卡卖两次)。
- **资金守恒(B1·CRITICAL)**:负欠隔离——已提现订单退款的差额落 `merchants.debt`、入账先抵欠、有负欠禁提现,杜绝"下单→秒提→退款"双向垫付/重复套现。
- **限量券(B2)**:下单即占额真限量(售罄拒单、关单释放、已付款永久占用),杜绝并发超发折扣。
- **异常单(B3)**:closed_then_paid 未结算单禁补发(防商户白送货),走退款。
- **存储型 XSS**:富文本净化器加固(协议白名单+实体解码),章节/资讯不再可注入脚本。
- 越权/信息泄露/输入校验:冻结商户商品不可购、知识/资源发货内容购前不泄露、CORS、上传 MIME、改密限流、投诉防刷、拉黑关在途单等。
- **结算幂等 DB 兜底**:恢复 `fund_logs uniq(order_id,type)`(退款佣金回冲改独立 type)。
