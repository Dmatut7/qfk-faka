# 更新日志

本项目版本遵循 [语义化版本](https://semver.org/lang/zh-CN/)。每个版本对应一个 git tag 与 GitHub Release。

## [v1.1.1] — 安全补丁

> ⚠ 使用 v1.1 的部署**强烈建议升级到本版**。

### 安全修复
- **(critical) 买家账号越权**:「我的订单」原按 `buyer_email` 匹配,而注册无邮箱验证 → 注册他人邮箱即可看其订单与已发货卡密。改为按 `buyer_id`(下单时凭买家令牌绑定)关联,不再按邮箱认领;历史游客单仍可在「订单查询」凭订单号+邮箱取卡。
- **(high)** `POST /admin/settings` 不再回显 `smtp_pass` 明文(与 GET 脱敏一致,防响应/日志/缓存泄漏)。
- **(high)** 买家并发同邮箱注册返回干净业务错误(`STATE_INVALID`)而非裸 500。
- **纵深防御**:SMTP 收发件地址去除 CR/LF 防命令/邮件头注入;卡密解密失败时记告警日志(密钥被改/丢失时让运维察觉)。

## [v1.1] — 特性补全

在 v1.0 核心闭环上补齐**全部预留特性**与半成品。

### 新增
- **发货邮件通知**:付款发卡后给买家邮箱送达卡密(零依赖 SMTP 客户端,fire-and-forget 绝不影响发货;后台「系统设置」可配 SMTP,密码脱敏存储)。
- **卡密落库加密**:配 `CARD_SECRET_KEY` 后导入即 AES-256-GCM 加密存储,发货/详情/列表透明解密;`secret_hash` 仍按明文去重;老明文卡向后兼容、可混存。
- **买家账号(可选)**:注册 / 登录 / 我的订单;游客流程零改动(不登录仍可下单 + 订单查询取卡)。
- **提现手续费**:按 `withdraw_fee_rate` 计算落库(此前为半成品)。
- **注册邀请码开关**:平台可开启「商户注册需邀请码」。

### 修复(节选)
支付回调跨单 `trade_no` 冲突静默丢单、Bearer 令牌 `?token=` 泄漏、订单查单密码哈希泄漏、退款无视冻结余额、投诉防订单枚举、`stock:reconcile` 负欠误报等。

## [v1.0] — 首个交付版本

多商户虚拟商品自动发卡平台,核心交易闭环完整且经测试。

### 能力
- **一卡一售(并发安全发卡)**:事务 + 行锁 + `FOR UPDATE SKIP LOCKED`,经 `pcntl_fork` 多进程真实并发验证。
- **支付回调三重保证**:验签防伪 · 金额一致防篡改 · 行锁内重查防重复(幂等)。易支付 `submit.php` 跳转 + 异步回调发卡。
- **资金安全**:全程 `bcmath`(`DECIMAL(10,2)`),禁浮点。结算抽佣 / 退款闭环 / 负欠隔离 / 提现 / 跨商户对账。
- **三端**:买家前台 · 商户后台 · 平台后台(橙色设计系统,移动端适配)。
- 优惠券·促销 / 投诉售后 / 资源下载防盗链 / 注册邀请码。
