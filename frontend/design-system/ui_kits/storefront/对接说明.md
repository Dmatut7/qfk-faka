# 买家前台 · 前后端对接说明

把现有原型接入真实后端,只需替换 4 个文件里的**模拟逻辑**为真实接口调用。下面按「页面 → 需要的接口 → 代码改哪里」列出。

> 约定:所有金额用「分」或两位小数字符串都行,前端 `PriceTag` 接收 number(元)。订单号字段名 `orderNo`,卡密数组字段名 `keys`。

---

## 1. 店铺首页 / 商品列表 — `StorefrontHome.jsx`

**需要的接口**
- `GET /api/shop` → 店铺信息:`{ name, intro, verified, avatar, cover, stats:{ products, deals, deposit } }`
- `GET /api/products?cat=<id>` → 商品列表:`[{ id, cat, name, desc, price, original, stock, sold, thumb, date }]`
- `GET /api/categories` → 分类:`[{ id, name }]`

**代码改哪里**
- 现在用 `window.MK_SHOP / MK_PRODUCTS / MK_CATEGORIES`(在 `data.js`)。改为在组件里 `useEffect` 拉取上面三个接口写入 state。
- `thumb` 当前是 emoji,接真实图片时传 `<img src={p.thumb}>`(`ProductCard / ProductListItem` 的 `thumb` 已支持 `<img>`)。

---

## 2. 商品详情 + 下单 — `ProductDetail.jsx`

**需要的接口**
- `GET /api/products/:id` → 商品详情(含 `detail` 长文案、实时 `stock`)
- `POST /api/orders` → 创建订单。
  - 请求:`{ productId, qty, email }`
  - 返回:`{ orderNo, status:'pending', product, qty, email, total }`

**代码改哪里**
- `submit()` 里现在直接 `onBuy({...})`。改为:校验邮箱后 `POST /api/orders`,拿到后端返回的 `orderNo` 再进入支付页。
- `App.jsx` 的 `buy()` 现在用 `mkOrderNo()` 本地造单号 —— **删掉**,改用后端返回的 `orderNo`。

---

## 3. 支付页 — `PaymentScreen.jsx`

**需要的接口**
- `GET /api/pay/methods` → 可用支付方式 `[{ id, name, desc, icon, tag }]`(可选,也可前端写死)
- `POST /api/pay` → 发起支付。
  - 请求:`{ orderNo, method }`
  - 返回:`{ payUrl }`(跳转收银台)或 `{ qrcode }`(展示二维码)
- `GET /api/orders/:orderNo` → 轮询订单状态,直到 `status` 变为 `paid`/`delivered`

**代码改哪里**
- `pay()` 现在是 `setTimeout(()=>onPaid(order),1400)` 模拟。改为:
  1. `POST /api/pay` → 跳转 `payUrl` 或弹二维码;
  2. 支付成功回调 / 轮询 `GET /api/orders/:orderNo`,`status==='delivered'` 时跳到取卡页。
- 卡密**不要**在前端生成 —— `App.jsx` 的 `mkKeys()` 删掉,卡密由后端在 `keys` 字段下发。

---

## 4. 订单查询 / 取卡 — `OrderLookup.jsx`

**需要的接口**
- `GET /api/orders/lookup?orderNo=<x>` 或 `?email=<x>` → 订单 + 卡密
  - 返回:`{ orderNo, status, product, qty, total, email, keys:[...] }`
  - **安全要点**:`status !== 'delivered'` 时,后端 **不要返回 `keys`**(前端 `CardKey locked` 会显示占位,不泄露)。

**代码改哪里**
- `search()` 现在调 `MK_SAMPLE_ORDER()` 造假数据。改为 `GET /api/orders/lookup`,把返回写入 `setResult`;查无订单时 `setError('未找到该订单')`。
- `window.MK_SAMPLE_ORDER`(在 `App.jsx`)删掉。

---

## 需要替换的「模拟点」清单

| 文件 | 函数/变量 | 改成 |
|---|---|---|
| `data.js` | `MK_SHOP / MK_PRODUCTS / MK_CATEGORIES` | 接口拉取 |
| `App.jsx` | `mkOrderNo()` | 用后端返回的 `orderNo` |
| `App.jsx` | `mkKeys()` | 删除,卡密由后端 `keys` 下发 |
| `App.jsx` | `MK_SAMPLE_ORDER()` | 删除,改真实查单接口 |
| `ProductDetail.jsx` | `submit() → onBuy` | `POST /api/orders` |
| `PaymentScreen.jsx` | `pay() 的 setTimeout` | `POST /api/pay` + 轮询状态 |
| `OrderLookup.jsx` | `search() 的 MK_SAMPLE_ORDER` | `GET /api/orders/lookup` |

## 状态字段对照(后端 `status` → 前端组件)
`OrderStatusBadge` / `CardKey` 直接吃这些值:
`pending`(待支付) · `paid`(已支付·发货中) · `delivered`(已发货,展示卡密) · `failed` · `refunded` · `closed`
